TAREFAS:

1) LINKAR DOMÍNIO AO DEPLOYMENT
- Vá em Deployments → Settings → Domains.
- Se hubpdf.pro estiver “Verified” mas NÃO vinculado, clique em “Link domain” e vincule hubpdf.pro ao deployment atual.
- Inclua também www.hubpdf.pro se disponível (opcional). 
- Aguarde a confirmação de “custom domain linked” e SSL emitido.

2) HEALTHCHECK
- Garanta um endpoint GET /healthz que responda 200 com JSON {"ok": true}.
- Se não existir, crie no app (FastAPI).

3) ROTA RAIZ / REDIRECTS
- Garanta que GET "/" responda 200. Se a home é /home, faça redirect 302 de "/" -> "/home".
- Adicione redirects:
  - HTTP → HTTPS (force HTTPS).
  - www.hubpdf.pro → https://hubpdf.pro (preferência por apex).
  OBS: implemente da forma mais simples possível (middleware ou configuração do servidor), sem quebrar rotas existentes.

4) TRUSTED HOSTS / CORS
- Em FastAPI, configure TrustedHostMiddleware para aceitar:
  ["hubpdf.pro", "www.hubpdf.pro", "hubpdf.replit.app", "localhost"]
- Se houver CORS, inclua https://hubpdf.pro e https://www.hubpdf.pro como allowed origins.
- Não abra CORS globalmente; somente para esses domínios (e localhost em dev).

5) RUN COMMAND E PORTA
- Confirme que o Run/Start do deployment é:
  uvicorn main:app --host 0.0.0.0 --port 5000
- Não mude a porta (5000).

6) ESTÁTICOS/TEMPLATES
- Verifique mounts de estáticos/templates para produção.
- Corrija caminhos relativos quebrados (ex.: /static/...).

7) RE-DEPLOY
- Após o domínio estar “Linked” e SSL emitido, faça re-deploy (Promote/Publish) e aguarde sucesso.
- Teste internamente acessos para:
  - https://hubpdf.pro/healthz  → deve retornar {"ok": true}
  - https://hubpdf.pro/        → deve carregar (ou redirecionar para /home)
  - https://hubpdf.pro/home   → deve carregar a Home
