REPLIT AI — Correção de login (usuário/e-mail não cadastrado)
Corrija o fluxo de login do projeto HubPDF conforme as instruções abaixo.

Objetivo:
Ao tentar logar com um e-mail ou usuário que não esteja cadastrado, o sistema deve exibir um ALERTA claro e um link de registro. O comportamento atual mostra apenas o texto "EMAIL NAO CADASTRAD." sem alerta visual.

---

1️⃣ **Comportamento desejado**
- Se o e-mail informado não existir no banco de dados (tabela `users`):
  - Exibir um alerta em destaque (ex.: toast, banner, ou div com classe de alerta).
  - A mensagem deve ser:
    ```
    ⚠️ E-mail não cadastrado. <a href="/auth/register" class="text-blue-600 hover:text-blue-500 font-medium underline">Clique aqui para se cadastrar</a>.
    ```
  - O alerta deve desaparecer automaticamente após alguns segundos **ou** ser fechado pelo usuário (botão “x”).

- Se o campo de e-mail estiver vazio, exibir:


Por favor, insira seu e-mail antes de continuar.


- Se o campo de senha estiver vazio:


Por favor, informe sua senha para continuar.


---

2️⃣ **Implementação**
- No handler da rota `/auth/login`:
- Após a busca:
  ```js
  const user = await db.query("SELECT * FROM users WHERE lower(email) = lower($1)", [email]);
  if (user.rowCount === 0) {
    return res.status(404).json({
      error: true,
      message: "E-mail não cadastrado. <a href='/auth/register' class='text-blue-600 hover:text-blue-500 font-medium underline'>Clique aqui para se cadastrar</a>."
    });
  }
  ```
- Retornar o HTML seguro (não escapado) no front, apenas para esse alerta.

- No front-end (HTML/JS):
- Verificar se a resposta do back-end contém `error: true`.
- Injetar a mensagem dentro de um container de alerta acima do formulário.
- Exemplo simples:
  ```js
  if (response.error) {
    const alertBox = document.getElementById("alert-box");
    alertBox.innerHTML = response.message;
    alertBox.classList.add("alert", "alert-warning");
    alertBox.style.display = "block";
  }
  ```
- Se o alerta for fechado ou após 10 segundos:
  ```js
  setTimeout(() => alertBox.style.display = "none", 10000);
  ```

---

3️⃣ **Estilo do alerta**
Use uma aparência simples, sem alterar o layout existente.  
Exemplo de CSS:
```css
.alert {
padding: 10px 15px;
background-color: #FFF3CD;
color: #856404;
border: 1px solid #FFEEBA;
border-radius: 5px;
margin-bottom: 10px;
font-size: 0.95em;
}


4️⃣ Garantir compatibilidade

A validação deve funcionar tanto para e-mail inexistente quanto para e-mail válido, mas senha incorreta.

As mensagens devem estar sempre em português com acentuação correta.

O fluxo de login funcional (redirecionar após login correto) não deve ser alterado.

5️⃣ Resumo
✔ Mostrar alerta visual quando o e-mail não estiver cadastrado.
✔ Incluir link “Clique aqui para se cadastrar”.
✔ Textos em português correto.
✔ Sem alterar design, menus ou layout do sistema.
